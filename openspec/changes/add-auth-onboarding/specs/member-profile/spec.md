# member-profile

## ADDED Requirements

### Requirement: 프로필 조회
시스템은 `GET /api/members/me`로 로그인한 회원의 프로필(닉네임, 성별, 출생연도, 키, 활동량, 목표 체중, 일일 칼로리 목표, 최신 체중, 온보딩 완료 여부)을 반환해야 한다(SHALL). 최신 체중은 weight_log의 가장 최근 기록에서 가져와야 한다(MUST).

#### Scenario: 내 프로필 조회
- **WHEN** 로그인한 회원이 프로필 화면에 진입하면
- **THEN** 저장된 프로필·목표와 최신 체중이 표시된다

### Requirement: 목표·프로필 수정
시스템은 키, 목표 체중, 활동량, 일일 칼로리 목표를 수정할 수 있어야 한다(SHALL). 키·목표 체중·활동량이 변경되면 새 제안 칼로리를 계산해 보여주되, 최종 저장값은 사용자가 확정한 값이어야 한다(MUST).

#### Scenario: 활동량 변경 후 재계산 제안
- **WHEN** 회원이 활동량을 LOW에서 HIGH로 변경하면
- **THEN** 새 제안 일일 칼로리가 표시되고, 회원이 확정한 값이 저장된다

#### Scenario: 칼로리 목표 직접 수정
- **WHEN** 회원이 일일 칼로리 목표만 직접 수정해 저장하면
- **THEN** 재계산 없이 입력한 값이 저장된다 (온보딩과 동일한 유효 범위 검증 적용)

### Requirement: 로그아웃 진입점
프로필 화면은 로그아웃 기능을 제공해야 한다(SHALL).

#### Scenario: 프로필에서 로그아웃
- **WHEN** 회원이 프로필 화면에서 로그아웃을 누르면
- **THEN** auth의 로그아웃 처리가 수행되고 로그인 화면으로 이동한다
